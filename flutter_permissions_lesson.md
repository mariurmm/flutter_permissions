# Permissions в мобильных приложениях (Flutter)

Практически любое мобильное приложение рано или поздно сталкивается с
необходимостью доступа к функциям устройства: камере, геолокации,
микрофону, галерее, уведомлениям и т.д.

Однако операционные системы не позволяют приложениям получать такой
доступ автоматически. Для защиты пользователя используется механизм
**permissions (разрешений)**.

В этом уроке разберём:

-   что такое permissions\
-   как они работают в iOS и Android\
-   как запрашивать их во Flutter\
-   лучшие практики работы с разрешениями

------------------------------------------------------------------------

# Что такое Permissions

**Permissions** --- это системные разрешения, которые пользователь
должен выдать приложению, чтобы оно получило доступ к определённым
возможностям устройства.

Например:

  Возможность   Permission
  ------------- ------------------
  Камера        Camera
  Геолокация    Location
  Фото          Photos / Storage
  Микрофон      Microphone
  Уведомления   Notifications

Когда приложение пытается получить доступ к одной из этих возможностей,
система показывает пользователю диалог:

> "Allow this app to access your camera?"

Пользователь может:

-   разрешить доступ
-   запретить доступ

И приложение **обязано корректно обработать оба варианта**.

------------------------------------------------------------------------

# Почему permissions важны

Permissions --- это не просто техническая деталь.

Они напрямую влияют на:

## Безопасность пользователя

Без permission любое приложение могло бы:

-   записывать звук
-   следить за местоположением
-   читать фотографии

Поэтому доступ к чувствительным данным строго контролируется системой.

## Политику магазинов приложений

И Apple, и Google внимательно проверяют использование разрешений.

Например:

-   нельзя запрашивать доступ к геолокации без объяснения
-   нельзя использовать камеру без явной причины
-   нельзя запрашивать лишние permissions

Нарушение этих правил может привести к **отклонению приложения в App
Store или Google Play**.

------------------------------------------------------------------------

# Как работают permissions

Общий жизненный цикл разрешения выглядит так:

1.  Приложение запрашивает permission
2.  Система показывает системный диалог
3.  Пользователь выбирает Allow или Deny
4.  Приложение получает статус разрешения

Статусы permission обычно бывают такими:

  -----------------------------------------------------------------------
  Статус                              Описание
  ----------------------------------- -----------------------------------
  granted                             разрешение выдано

  denied                              пользователь отказал

  permanently denied                  пользователь запретил и отключил
                                      повторный запрос

  restricted                          ограничено системой
  -----------------------------------------------------------------------

# Permissions во Flutter

Во Flutter чаще всего используют пакет:

**permission_handler**

Он позволяет:

-   проверять статус permission
-   запрашивать permission
-   открывать настройки приложения

Пакет поддерживает:

-   Android
-   iOS
-   Web
-   macOS

------------------------------------------------------------------------

# Установка permission_handler

Добавляем зависимость:

``` yaml
dependencies:
  permission_handler: ^11.0.0
```

После этого выполняем:

``` bash
flutter pub get
```

------------------------------------------------------------------------

# Настройка Android permissions

В Android разрешения объявляются в файле:

    android/app/src/main/AndroidManifest.xml

Например, для камеры:

``` xml
<uses-permission android:name="android.permission.CAMERA"/>
```

Для геолокации:

``` xml
<uses-permission android:name="android.permission.ACCESS_FINE_LOCATION"/>
```

------------------------------------------------------------------------

# Настройка iOS permissions

В iOS необходимо добавить описание разрешения в файл:

    ios/Runner/Info.plist

Например для камеры:

``` xml
<key>NSCameraUsageDescription</key>
<string>We need camera access to take photos</string>
```

Это сообщение **показывается пользователю в системном диалоге**.

------------------------------------------------------------------------

# Проверка permission

Перед тем как запрашивать доступ, лучше проверить его статус.

``` dart
var status = await Permission.camera.status;

if (status.isGranted) {
  print("Camera permission already granted");
}
```

------------------------------------------------------------------------

# Запрос permission

Если разрешение не выдано, его можно запросить:

``` dart
var status = await Permission.camera.request();

if (status.isGranted) {
  print("Permission granted");
}
```

------------------------------------------------------------------------

# Обработка отказа пользователя

Очень важно корректно обработать ситуацию, когда пользователь отказал.

``` dart
if (status.isDenied) {
  print("User denied permission");
}
```

Если пользователь запретил разрешение **навсегда**, приложение должно
предложить открыть настройки.

``` dart
if (status.isPermanentlyDenied) {
  openAppSettings();
}
```

------------------------------------------------------------------------

# Лучшая практика: запрашивать permission в нужный момент

Плохая практика:

❌ запрашивать все permissions при запуске приложения

Хорошая практика:

✅ запрашивать permission **когда пользователь пытается использовать
функцию**

Пример:

-   пользователь нажал кнопку **"Сделать фото"**
-   приложение запрашивает **Camera permission**

Это повышает вероятность того, что пользователь согласится.

------------------------------------------------------------------------

# UX-подход: предварительное объяснение

Перед системным диалогом лучше показать **свой экран объяснения**.

Например:

> Чтобы распознать растение, приложению нужен доступ к камере.

После этого --- кнопка:

**Allow camera**

И уже после неё вызывается системный запрос permission.

------------------------------------------------------------------------

# Частые ошибки

## Запрашивать слишком много permissions

Это снижает доверие пользователя.

## Не обрабатывать отказ

Если пользователь отказал, приложение **не должно падать**.

## Не объяснять причину доступа

Пользователь должен понимать:

> зачем приложению нужен доступ.

------------------------------------------------------------------------

# Итог

Permissions --- важная часть мобильной разработки.

Нужно помнить три ключевых правила:

1️⃣ Запрашивайте только необходимые permissions\
2️⃣ Делайте это в момент использования функции\
3️⃣ Всегда обрабатывайте отказ пользователя

Во Flutter работа с разрешениями значительно упрощается благодаря пакету
**permission_handler**, который предоставляет единый API для Android и
iOS.
